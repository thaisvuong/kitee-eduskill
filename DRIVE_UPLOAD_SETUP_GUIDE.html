Hướng dẫn setup upload Google Drive cho KientreAAA

Tài liệu này dành cho anh và người khác dùng webapp KientreAAA để tạo file Word rồi upload lên Google Drive bằng file .json OAuth.

Kết quả test thật:
  • Đã xác minh bằng file OAuth JSON lưu tại webapp.
  • Đã upload thành công 1 file .docx và 1 bản Google Docs.
  • Folder test đã dùng: folder Output trong Drive của dự án.

1. Cần chuẩn bị gì

Phần bắt buộc

  • Webapp KientreAAA đang chạy local tại http://localhost:3100
  • Một file OAuth Google dạng .json
  • Một folder Google Drive để nhận file upload

Phần app sẽ làm

  • Tạo file Word local trước
  • Upload file .docx lên Drive
  • Tạo thêm 1 bản Google Docs
  • Hiện link trong tab Kết quả

2. Cách setup cho chính anh

  1. Mở http://localhost:3100
  2. Vào Cài đặt
  3. Tìm khối Google OAuth JSON
  4. Dán hoặc upload file .json
  5. Bấm Lưu & làm mờ
  6. Bấm Tạo link cấp quyền
  7. Mở link Google consent và đồng ý quyền
  8. Khi trình duyệt nhảy sang localhost:1 và báo lỗi, copy toàn bộ URL trên thanh địa chỉ
  9. Quay lại webapp, dán URL đó vào ô URL redirect sau khi Google báo lỗi localhost:1
  10. Bấm Lưu quyền Drive
  11. Bấm Kiểm tra Drive
Ghi chú: Trang localhost:1 lỗi là bình thường. Điều quan trọng là phải copy đúng toàn bộ URL redirect có tham số code=...

3. Nhập folder upload ở đâu

  1. Vào đúng module anh muốn dùng, ví dụ Soạn quiz theo chuyên đề hoặc Review
  2. Bấm Cài đặt module
  3. Tại ô Folder Drive output, dán link folder Drive hoặc dán thẳng folder ID
  4. Bật checkbox Upload .docx lên Drive + convert Google Docs

Folder test hiện tại

Dùng folder Output trong Drive của dự án, hoặc dán link/ID folder riêng của anh.

ID app sẽ tự lấy từ link https://drive.google.com/drive/folders/....

4. Cách setup cho người khác

Người khác cũng làm y hệt, nhưng phải dùng tài khoản Google của họ và folder Drive của họ.

Phần Dùng chung được không? Ghi chú
Code webapp Có Dùng chung repo/app
OAuth JSON Không Mỗi người phải dùng file riêng
Drive folder Không Mỗi người nên có folder riêng
Link consent Không Phải tạo lại theo tài khoản đang đăng nhập

5. Cách test upload sau khi setup

  1. Tạo 1 job nhỏ như review 1 file txt/docx hoặc soạn 1 bài ngắn
  2. Đợi job xong
  3. Vào tab Kết quả
  4. Kiểm tra có hiện:
    • link Google Docs
    • link tải file .docx
  5. Mở folder Drive đã cấu hình để kiểm tra file thật

7. Các lỗi thường gặp

  • 403 access_denied — tài khoản Google chưa nằm trong Test users của OAuth app
  • Google Drive API not enabled — chưa bật Google Drive API trong project Google Cloud
  • credential has no access_token/token/refresh_token yet — mới lưu client JSON, chưa làm bước consent + paste redirect URL
  • Upload chạy xong nhưng tab Kết quả chưa hiện file — cần kiểm tra lại cấu hình folder Drive và trạng thái checkbox upload trong đúng module
